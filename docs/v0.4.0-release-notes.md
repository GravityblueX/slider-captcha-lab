# v0.4.0 Release Notes

主题：真实 Chrome CDP 会话接入。

## 新增

- 新增 `src/chrome_session.py`：
  - 支持 `browser.connect_existing_chrome` / `browser.cdp.endpoint`；
  - 可复用本机授权 Chrome 的当前页面、登录态、Cookie 和扩展环境；
  - 默认仍保留原来的托管 Chromium 路径。
- 新增 `examples/attached_chrome_profile.json`，作为接入已有 Chrome 的最小 Profile 示例。
- 新增 `scripts/start-authorized-chrome-cdp.ps1`，用于启动独立 user-data-dir 的授权 Chrome 调试会话。
- 新增 `scripts/cdp_attach_smoke.py`，用临时 Playwright Chromium 验证 CDP attach 链路，不接触日常 Chrome 配置。

## 改进

- `src/page_probe.py` 和 `src/cdp_diagnostics.py` 统一走 `chrome_session`；
- 页面探测和 CDP 诊断 JSON 增加 `browser_session` 字段；
- 综合报告中心可以区分“真实 Chrome”和“托管 Chromium”；
- README 和 CDP 文档补充已打开 Chrome 的接入流程。

## 安全边界

- 本功能仅用于本地、自有或明确授权页面的 QA、诊断和报告留档；
- 不实现验证码自动求解；
- 不提供真实站点风控、验证码、访问控制或反爬虫绕过逻辑；
- Cookie 报告仍只记录名称和数量，不记录 Cookie 值。
